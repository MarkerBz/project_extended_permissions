PROJECT EXTENDED PERMISSIONS ================================:

- Новый функционал ДОПОЛНЯЕТ/ИЗМЕНЯЕТ, а НЕ ЗАМЕНЯЕТ стандартные
    правила доступа по Проектам, то есть внутри формы Проекта
    настройка "Конфиденциальность" продолжает работать и её
    можно устанавливать в положение "Видимый по всем сотрудникам".

- У каждого Пользователя, который будет работать с функционалом
    Проектов, в его форме необходимо заполнить поле "Проект":
        - "Проект: Пользователь"
        или
        - "Проект: Менеджер"
	Новый функционал разграничения доступа в основном работает
    с пользователями типа "Проект: Пользователь".

- Этот модуль деактивирует три стандартных глобальных правила доступа:
	- "Задача - Project/Task: multi-company",
	- "Проект - Project: multi-company",
	- "Аналитический счёт - Analytic multi company rule".

	После удаления данного модуля, эти правила снова активируются.

- У каждого Пользователя, который будет работать с функционалом
    Проектов, в его форме необходимо установить какой-нибудь из
    двух флагов доступа:
        - "Проект: право чтения и изменения ВСЕХ задач и вопросов",
        - "Проект: право чтения и изменения НАЗНАЧЕННЫХ на пользователя
            задач и вопросов".
